## Admin 项目图片目录支持子文件夹

**需求**：修改/新建项目时，图片默认只能放 `public/images/` 扁平目录；改为 `public/images` 为图片根目录，允许在 `public/images` 下新建子文件夹作为单个项目的图片。

**改动**（`src/admin.jsx`）：
- `MediaFileInput` 新增 `folder` 参数：图片上传路径由 `/images/<file>` 改为 `/images/<folder>/<file>`（视频仍为 `/videos/`）
- 项目模板新增 `imageFolder: ''` 字段
- 项目卡片新增「图片目录」输入字段（占位示例 `brand-visual`）
- 项目媒体图片 `MediaFileInput` 传入 `folder={proj.imageFolder}`，提示文案随目录动态变化
- 顶部「媒体素材存放说明」更新为支持子文件夹的路径规则

**兼容性**：现有项目无 `imageFolder` 字段 → 视为空字符串 → 图片仍存 `public/images/`（扁平），无破坏性。生产端 Projects.jsx / ProjectModal.jsx 直接以路径为 `src`，子文件夹路径天然支持，无需改动。

**构建验证**：`npm run build` 通过（454 modules）。

## Admin 项目素材目录重构：图片+视频统一按项目分目录

**需求**：视频和图片放到一个文件根目录，以项目区分，不要用素材类型区分。

**改动**（`src/admin.jsx`）：
- `MediaFileInput` 逻辑重构：`folder` 参数存在时，图片与视频统一生成路径 `/projects/<folder>/<file>`；无 `folder` 时图片走 `/images/`、视频走 `/videos/`（全局素材维持原样）
- 项目字段 `imageFolder` 重命名为 `projectDir`，语义从「图片子文件夹」变为「项目素材根目录」
- 项目卡片「图片目录」字段 →「项目素材目录」字段
- gallery 图片与视频上传均传入 `folder={proj.projectDir}`，统一归入 `/projects/<dir>/`
- 新增「封面图」输入（归入项目目录）
- 顶部「媒体素材存放说明」更新：项目素材统一 `public/projects/<项目目录>/`，全局素材（头像/Hero视频/图标/二维码）仍走 images/videos

**范围边界**：仅项目内素材（封面图、gallery 图片、gallery 视频）归入 projects 目录；全局素材（头像、`/videos/hero-bg.mp4`、能力图标、公众号二维码）维持 `public/images/`、`public/videos/` 不变。

## Admin 媒体上传真正落地 public（修复预览不显示）

**问题**：后台「选择文件」只是生成路径字符串（如 `/projects/xxx/cover.jpg`），并未把文件写入 `public/`，导致预览 `<img src="/projects/xxx/file.jpg">` 指向磁盘不存在的文件 → 404 → 不显示。

**修复**：
- `vite.config.js` 新增 `mediaUploadPlugin()`：dev server 中间件 `POST /__media-upload` 接收 `{dir, name, dataUrl}`，Base64 解码后写入 `public/<dir>/<name>`；仅允许 `projects/images/videos` 根、禁止 `..`，仅 dev server 生效（admin 是本地工具，不进生产构建）
- `MediaFileInput.handleFile` 改为异步：FileReader 转 dataURL → fetch 上传 → 成功后 `onChange(返回路径)`；大文件(>15MB)或失败回退路径模式并 `alert` 提示手动放置；`finally` 清空 input 允许重选同名
- 封面图输入新增预览缩略图

**验证**：`POST /__media-upload` 返回 `{ok:true, path}` 且文件真实写入 public；dev server 重启后生效（task zz7pTH）。

**注意**：上传仅在 dev server（admin 本地工具）生效；部署时 public 下文件随构建一起上线。

## ProjectModal Lightbox 滚轮缩放

**新增**：放大后滚轮上下滚动放大/缩小图片。

**改动**（`src/components/ProjectModal.jsx`）：
- 新增 `zoomScale` 状态（默认 1）+ `zoomScaleRef`；`openZoom(src)` 打开时重置 scale=1

- Esc 键优先级：scale>1 → 先重置 scale=1；否则关 Lightbox；否则关 modal
- `GalleryItem` 的 `onZoom` 由 `setZoomedImage` 改为 `openZoom`（打开即重置缩放）

## Admin 上传图片自动压缩（sharp）

**问题**：上传素材太大，后台编辑卡顿 + 线上加载慢。

**方案**：上传时自动压缩，不需要用户手动处理。

**改动**：
- 项目新增 `devDependencies`: `sharp`（Node.js 图片处理库，零依赖纯 JS）
- `vite.config.js` `mediaUploadPlugin` 重构：
  - 新增 `compressImage(buffer, mimeType)` 异步函数：JPEG → mozjpeg q82，PNG → palette + level9，WebP → q82；统一 resize 长边 ≤ 1920px（不放大），压缩失败退回原图
  - 中间件 handler 改为 async，请求体收集改用 `for await` 流式（替代 `req.on('data'/'end')` 回调）
  - 压缩后在终端打印 `📷 [压缩] 文件名  原大小 → 新大小  (压缩率%)`
  - 非图片文件（视频/SVG/GIF）原样写入，不压缩
- `MediaFileInput` 移除前端 15MB 限制（服务器端 60MB 上限已足够）

**效果预期**：典型 4000×3000 手机照片 ~4MB → 1920px + q82 → ~200KB（≈95% 压缩率），后台预览不再卡，线上秒加载。

**构建验证**：`npm run build` 通过（454 modules）。
