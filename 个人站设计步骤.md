## 第七步 — 移动端适配

把以下指令发给智能体：

---

网站目前是 PC 端 1700px 版心，请帮我做完整的移动端适配。要求：

### 7.1 断点策略

- **桌面端**：≥ 1024px，版心 1700px，保持现有布局
- **平板端**：768px - 1023px，版心收窄至 100%，适当减少 padding
- **移动端**：≤ 767px，单列堆叠布局

### 7.2 具体调整

- **导航栏**：移动端改为汉堡菜单，点击展开全屏抽屉式导航
- **Hero 首屏**：视频背景降级为静态封面图（移动端不加载视频，节省流量）；大标题字号缩小但保持视觉冲击
- **个人经历模块**：头像和介绍从左右布局改为上下堆叠
- **精选项目卡片**：从多列网格改为单列纵向滚动，卡片高度适配移动端
- **个人优势卡片**：从多列改为单列或双列
- **底部联系模块**：按钮改为全宽堆叠
- **项目详情弹窗**：移动端改为全屏页面，关闭按钮固定顶部
- **所有字号**：用 clamp() 实现流式缩放，不要硬切
- **GSAP 动画**：移动端简化或关闭 parallax，保留基本进场动画
- **触摸交互**：确保所有 hover 效果在触摸设备上有对应的 active/tap 反馈

### 7.3 性能

- 移动端不加载 Hero 视频，用 poster 图替代
- 图片用 `<picture>` + srcset 提供移动端小图
- 移动端关闭自定义光标（如果有的话）
- 检查移动端滚动性能，必要时降低动画复杂度

## 第九步 — 构建 & 部署

### 零成本部署方案推荐：Cloudflare Pages

**为什么选 Cloudflare Pages：**
- 完全免费，无流量限制
- 全球 CDN 加速，国内访问速度相对稳定
- 连接 GitHub 自动部署：push 代码即上线，后期新增作品只需改代码 push 即可
- 自定义域名免费送 SSL 证书
- 支持 Vite / React 项目，零配置

**部署步骤：**

1. **代码推到 GitHub**
   - 在项目根目录确保有 `.gitignore`（排除 `node_modules/`、`dist/`）
   - 创建 GitHub 仓库（private 即可）
   - `git init && git add . && git commit -m "init" && git remote add origin <仓库地址> && git push -u origin main`

2. **连接 Cloudflare Pages**
   - 注册 / 登录 https://dash.cloudflare.com → Workers & Pages → Create application → Pages
   - 选择「Connect to Git」→ 授权 GitHub → 选择仓库
   - 构建配置：
     - Framework preset: **Vite**
     - Build command: `npm run build`
     - Build output directory: `dist`
     - Node version: 18（环境变量加 `NODE_VERSION=18`）
   - 点 Deploy，首次构建约 1-2 分钟

3. **后期更新作品**
   - 本地改代码 / 加项目 → `git push` → Cloudflare 自动重新构建部署
   - 无需任何手动操作

4. **绑定自定义域名（可选）**
   - Pages 项目 → Custom domains → Add domain
   - 按提示添加 CNAME 记录
   - SSL 自动签发，无需操心

**备选方案（如果 Cloudflare 访问不稳定）：**

| 方案 | 免费额度 | 国内速度 | 自动部署 | 备注 |
|------|---------|---------|---------|------|
| **Vercel** | 免费够用 | 偶尔慢/被墙 | git push 即部署 | 体验最好，但国内不稳定 |
| **Netlify** | 100GB/月 | 一般 | git push 即部署 | 老牌方案 |
| **GitHub Pages** | 无限 | 较慢 | git push 即部署 | 仅静态，最简单 |

> **建议**：先用 Cloudflare Pages 部署，如果国内访问不理想再切 Vercel。两者都是 git push 自动部署，切换成本极低。
